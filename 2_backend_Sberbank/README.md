# Преобразование витрины режимов работы сотрудников

Решение задачи по преобразованию данных о режимах работы сотрудников Банка из формата с периодами в покадровый формат.

## Описание задачи

Необходимо преобразовать витрину с режимами работы сотрудников из исходного формата (Таблица 1) в целевой формат (Таблица 3) по следующим правилам:
- Диапазон дат: 01.09.2020 – 31.12.2020
- Для каждого сотрудника и каждой даты определить флаг `to_be_at_office`:
  - `1` — сотрудник в офисе
  - `0` — сотрудник на дистанционной работе
  - `NULL` — выходной день или нет активного режима

## Стек технологий

- **Python 3.11+** — чистый Python без внешних зависимостей
- **unittest** — модульное тестирование
- **flake8** — линтер (wemake-python-styleguide)
- **isort** — сортировка импортов
- **Docker** — контейнеризация

---

### Способ 1: Локальный запуск

1.  **Клонируйте репозиторий и перейдите в папку проекта:**
    ```bash
    git clone <ссылка-на-репозиторий>
    cd work-schedule-transformer
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    
    *На Windows:*
    ```bash
    python -m venv venv
    venv\Scripts\activate
    ```
    
    *На Linux/macOS:*
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    ```
    
    *После активации в начале строки терминала должно появиться `(venv)`.*

3.  **Установите необходимые библиотеки (для разработки и линтинга):**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Запустите преобразование данных:**
    ```bash
    python src/main.py
    ```
    
    В консоли появится информация о сформированных витринах, проверка контрольных точек и первые 20 строк результата.

5.  **Запустите тесты (опционально):**
    ```bash
    python -m unittest discover src/
    ```

6.  **Проверьте стиль кода (опционально):**
    ```bash
    flake8 src/
    isort --check-only src/
    ```

### Способ 2: Запуск через Docker

1.  **Соберите Docker-образ:**
    Откройте терминал в папке `work-schedule-transformer` и выполните команду:
    ```bash
    docker build -t work-schedule-transformer .
    ```
    
    Эта команда создаст образ с именем `work-schedule-transformer` согласно инструкциям в файле `Dockerfile`.

2.  **Запустите контейнер:**
    ```bash
    docker run --rm work-schedule-transformer
    ```
    
    Флаг `--rm` автоматически удалит контейнер после завершения работы.

3.  **Запустите с тестами (опционально):**
    ```bash
    docker run --rm work-schedule-transformer python -m unittest discover src/
    ```
